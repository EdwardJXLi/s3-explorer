# S3 Explorer

A lightweight stateless web UI for S3-compatible object storage. It is designed for one configured storage instance and works with providers such as MinIO, Cloudflare R2, AWS S3, Backblaze B2, and other S3-compatible endpoints.

## Features

- Browse multiple configured buckets with breadcrumb, path, and filter navigation.
- View, download, upload, rename, move, and delete files and folder prefixes.
- Create folder prefixes.
- View standard and custom object metadata.
- Generate direct presigned download URLs with selectable expiry.
- Optional proxy downloads through the app.
- OIDC/OAuth2 login with PKCE or simple username/password authentication.
- Viewer/admin roles with email and configurable OIDC role mapping.
- No database or Redis required; sessions are signed HTTP-only cookies.
- Docker-ready single container.

## Quick Start

```bash
cp .env.example .env
npm install
npm run build
npm start
```

Open `http://localhost:8080`.

For local frontend development:

```bash
npm run dev
```

The Vite dev server runs on `http://localhost:5173` and proxies API calls to `http://localhost:8080`.

## Docker

```bash
docker build -t s3-explorer .
docker run --rm -p 8080:8080 --env-file .env s3-explorer
```

## Configuration

### Branding

- `APP_NAME`: name shown on the login screen, sidebar, and browser title. Defaults to `S3 Explorer`.
- `APP_ICON_URL`: optional image URL for the app mark. Use a small square PNG, SVG, or favicon-style asset.
- `APP_DEFAULT_THEME`: `light` or `dark`. Defaults to `light`; users can still toggle and persist their own choice.

### Storage

Use `STORAGE_*` variables so the deployment stays provider-neutral:

- `STORAGE_ENDPOINT`: S3-compatible endpoint, for example `http://minio:9000` or an R2 endpoint.
- `STORAGE_REGION`: signing region. Many compatible providers accept `us-east-1`.
- `STORAGE_ACCESS_KEY_ID`, `STORAGE_SECRET_ACCESS_KEY`, `STORAGE_SESSION_TOKEN`: credentials.
- `STORAGE_FORCE_PATH_STYLE`: defaults to `true`, which is commonly needed for MinIO.
- `STORAGE_BUCKETS`: comma-separated bucket allowlist shown in the UI.
- `STORAGE_DEFAULT_BUCKET`: bucket selected first.
- `STORAGE_ALLOW_BUCKET_LIST`: set `false` when credentials cannot list all buckets.
- `STORAGE_MAX_UPLOAD_BYTES`: max upload size handled by the app. Set `0` to disable the app-level upload limit.

### Authentication

Set `AUTH_MODE` to:

- `simple`: username/password from environment variables.
- `oidc`: OIDC/OAuth2 authorization code flow with PKCE.
- `none`: disables login. Intended only for trusted local or reverse-proxy-protected deployments.

Common settings:

- `SESSION_SECRET`: required in production. Use a long random value.
- `SESSION_TTL_SECONDS`: signed cookie session lifetime.
- `ADMIN_EMAILS`: comma-separated emails that receive admin permission.

Simple auth:

- `SIMPLE_AUTH_USERNAME`
- `SIMPLE_AUTH_PASSWORD`
- `SIMPLE_AUTH_EMAIL`
- `SIMPLE_AUTH_ADMIN`

OIDC:

- `OIDC_ISSUER_URL`
- `OIDC_CLIENT_ID`
- `OIDC_CLIENT_SECRET`: optional for public clients.
- `OIDC_REDIRECT_URI`: defaults to `${PUBLIC_BASE_URL}/api/auth/oidc/callback`.
- `OIDC_SCOPES`: defaults to `openid email profile`.
- `OIDC_EMAIL_CLAIM`, `OIDC_NAME_CLAIM`
- `OIDC_ROLE_CLAIM`: defaults to `roles`.
- `OIDC_ADMIN_ROLE_VALUES`: comma-separated role values that grant admin.
- `OIDC_EXTRA_AUTHORIZE_PARAMS`: comma-separated `key=value` pairs.

Users are viewers unless their email is in `ADMIN_EMAILS` or their configured OIDC role claim matches `OIDC_ADMIN_ROLE_VALUES`.

### Downloads

By default, downloads use direct presigned URLs generated by the storage backend. Set `ENABLE_PROXY_DOWNLOADS=true` to allow the UI to route downloads through the app at `/api/download`.

`DEFAULT_PRESIGN_SECONDS` controls the normal download button expiry. `MAX_PRESIGN_SECONDS` caps custom presigned URLs.

## Notes

Folder operations work on prefixes. Moving or deleting a folder lists every object under that prefix, then copies or deletes those objects in batches. Very large prefix moves can take time and depend on provider copy limits.
